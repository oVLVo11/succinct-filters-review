## 2. 必答问题记录

### 2.1 论文研究对象是什么？

Day 1 初步回答：

论文研究动态 membership data structures，包括精确的 dictionary 和允许误报的 filter。重点是集合大小未知时，空间使用能否依赖当前集合大小 `n`，同时支持高效插入和查询。

待核查：

- 正式模型是否允许删除；
- insert 重复元素时如何处理。

### 2.2 “unknown sizes” 具体未知的是什么？

Day 1 初步回答：

未知的是最终集合大小或一个紧的容量上界 `N`。当前 `n` 并非不可知，结构可以随着插入增长而调整空间。

### 2.3 为什么不能预先设置一个足够大的容量？

Day 1 初步回答：

如果预先设置过大的 `N`，传统结构会长期占用按 `N` 分配的空间；当实际 `n` 远小于 `N` 时，空间浪费严重。实际应用中最大容量常常只能保守估计，因此该问题具有现实意义。

### 2.4 论文主要结果分别是什么？

Day 1 初步回答：

- Dynamic filter：空间 `(1 + o(1)) n (log(1/epsilon) + log log n)`，插入和查询以高概率最坏情况常数时间。
- Dynamic dictionary：空间 `(1 + o(1)) n (log(u/n) + v + O(log log log u))`，插入和查询以高概率最坏情况常数时间。

待核查：

- 正式定理参数范围；
- 高概率事件定义；
- `epsilon` 的取值范围。

### 2.5 filter 与 dictionary 的结果有什么区别？

Day 1 初步回答：

dictionary 精确保存 membership/key-value 信息，不允许任何错误；filter 允许非成员查询误报 YES，但不允许成员查询误报 NO。filter 的空间主项与 `log(1/epsilon)` 相关，dictionary 的空间主项与 `log(u/n)` 和 value 长度 `v` 相关。
