# 阶段增长过程（与 8 元素示例一一对应）

> 来源：§1.4、§3.1、Claim 13。  
> 与 `construction-notes.md` §6、`core-components.md` 同步。  
> 横轴 `i` 指 **`i★=⌈log n⌉`（算法控制）**，不是证明段半开区间的自由变量。  
> 教学示例：不虚构真实哈希碰撞；不能用于 FP/空间/字面常数 10 的证明。

## 2. 每个 `n`：新元素去向 / 活跃查询结构 / 迁移在搬谁

```text
n=1  i=0
  新 → D0
  查 → D0, T0
  迁 → （无 T_{-1}/D_{-1}）

n=2  i=1
  新 → D1
  查 → D0, D1, T0, T1
  迁 → T0/D0 → T1/D1 ；推进 destroy 旧、init T2/D2

n=3  i=2
  新 → D2
  查 → D1, D2, T1, T2
  迁 → T1/D1 → T2/D2 ；推进 destroy、init T3/D3

n=4  i=2
  新 → D2
  查 → D1, D2, T1, T2
  迁 → 继续清空 T1/D1 ；趋向下一层 T3/D3 就绪

n=5  i=3
  新 → D3
  查 → D2, D3, T2, T3
  迁 → T2/D2 → T3/D3 ；推进 destroy、init T4/D4

n=6  i=3
  新 → D3
  查 → D2, D3, T2, T3
  迁 → 继续搬 T2/D2

n=7  i=3
  新 → D3
  查 → D2, D3, T2, T3
  迁 → 同上

n=8  i=3
  新 → D3
  查 → D2, D3, T2, T3
  迁 → 继续；按 Claim 13，幂次边界上旧层应迁完、下一层已 init
```

## 3. 单阶段内部（以 stage 3：`n=5..8` 为例）

```text
+------------------------------------------------------------------+
|  stage 3（i=3）                                                    |
|  新前缀 ──► 写入 D3                                                |
|                                                                  |
|  每次 Insert 重复 10 次:                                           |
|    T2 --decrement--> 扩展两位写入 T3                               |
|    D2 --decrement--> |y|>3 ? D3 : T3                               |
|    空则 destroy(T2/D2)；已销毁则 initialize(T4/D4)                 |
+------------------------------------------------------------------+
```

---

## 4. 与“预留超大 N / 一次性重建”对比

| 策略 | 在 n=1..8 上会怎样 | 问题 |
|---|---|---|
| 预分配容量 8 | 一开始就按 8 占满 | n=2 时已浪费 |
| n=4 时整体重建到 8 | 单次插入 `Θ(n)` | 最坏时间炸；filter 无原键可重哈希 |
| 本文 | 每步新键进 `D_i` + 常数步搬旧 | 查询结构数恒为常数 |

## 5. 读图检查清单

1. `n=5` 时新元素进哪个结构、查询活跃是哪四个？
2. 是否误以为 `T_i` 接收所有新插入？（否 → `D_i`）
3. “10 次”是否被理解成“搬迁 10 个元素”？（否 → 重复维护过程 10 遍）
4. 是否把本图当成半开区间存储不变式的证明？（否 → 仅 `i★` 控制流）
