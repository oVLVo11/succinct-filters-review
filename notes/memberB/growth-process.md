# 阶段增长过程

> 来源：论文 §1.4、§3.1、Claim 13。  

---

## 横轴：插入次数 `n` 与阶段 `i = ⌈log n⌉`

```text
n:  1······2······4······8······16······32······
i:   0/1    1     2     3      4       5
      |←stage i→|←stage i+1→|
      规模约翻倍，容量目标约 2^i → 2^{i+1}
```

阶段 `i`：从第 `2^{i-1}+1` 次插入到第 `2^i` 次插入（文中表述）。

---

## 每个阶段内部在做什么

```text
                    阶段 i（n 从 ~2^{i-1} 增到 ~2^i）
+------------------------------------------------------------------+
|  新插入的长前缀 ──► 写入 D_i（容量导向 2^i）                      |
|  短前缀相关逻辑 ──► 经由 T_{i-1} → T_i 的扩展迁移                 |
|                                                                  |
|  后台（每次 Insert 常数步）:                                      |
|    D_{i-1} --decrement--> 迁入 D_i 或 T_i                         |
|    T_{i-1} --decrement--> 扩展两位写入 T_i                        |
|    清空后 destroy 旧结构；initialize 下一阶段 T_{i+1}, D_{i+1}     |
+------------------------------------------------------------------+
```

阶段结束时的目标快照：

- 所有已插入串落在 `D_i, T_i`；
- `D_{i-1}, T_{i-1}` 已销毁；
- `D_{i+1}, T_{i+1}` 已初始化，为下一翻倍做准备。

## 空间（为何不预留超大 `N`）


| 策略            | 行为                 | 问题                   |
| ------------- | ------------------ | -------------------- |
| 预分配容量 `N`     | 一开始就按 `N` 占空间      | `n ≪ N` 时浪费          |
| 一次性重建到 `2n`   | 某次插入触发 `Θ(n)` 工作   | 最坏时间炸裂；filter 还丢了原键  |
| 本文分阶段 + 去均摊迁移 | 空间跟当前 `n`；每次插入搬常数步 | 需精心设计仍可查询、且 succinct |

## 与 PSW 旧方案的对比

```text
PSW 思路（简化）:
  多阶段过滤器 / 或短串复制成长串 → 丢进一个 Raman-Rao 字典
  插入：期望均摊 O(1)

本文:
  短串 → Truth table
  长串 → 新 membership（data block，冗余更小）
  容量翻倍 → in-place、可查询的后台迁移
  插入：最坏 O(1) whp，且 log log n 项前导常数钉死
```
