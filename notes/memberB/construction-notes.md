## 1. 元素如何经全局哈希变成二进制串

原文：§3.1。

1. 取全局哈希 `h : [u] → [u^{c_2}]`，要求 `(c_1 log u)`-wise independent（Thm 7）。
2. 对每个 key `x`，`h(x)` 视为足够长的二进制串。
3. 第 `n` 次插入时令 `i = ⌈log n⌉`，只把 `h(x)` 的**前 `ℓ_i` 位**写入 prefix-matching 结构（式 (2)）。
4. Filter 不再保存原始 `x`；之后所有正确性/误报分析都在这些前缀串上进行。

**教学标注**：下面 8 元素例子中的比特串是为演示状态变化而指定的示意前缀，**不是**真实 `h` 的输出，也不代表碰撞概率。

## 2. 前缀长度公式（式 (2)）逐项含义

```text
ℓ_i = i + log(1/ε) + log i + log log log u + 2
```

| 项 | 作用（Day 2 理解） | 原文依据 |
|---|---|---|
| `i` | 随阶段增大，前缀变长；与“未知规模/多阶段并存”相关的位数 | §1.4 末、§3.1 |
| `log(1/ε)` | 主误报率预算，对应经典 filter 每键约 `log(1/ε)` 位 | §3.1 并合计算前 |
| `log i` | 把各阶段误报压到可对 `i` 求和；文中并合出现 `1/(i · …)` | §3.1 紧接式 (2) |
| `log log log u` | 给底层 membership / 冗余留出余量，使总空间里该项进 `O(·)` | §1.4、Lemma 11 空间 |
| `+2` | 常数松弛，使并合严格 `< ε` | §3.1：并合得 `ε · ∑ 1/(4 i log log u) < ε` |

非成员误报（pairwise / 并合，§3.1）：

```text
Pr[FP] ≤ ∑_{i=1}^{log u}  2^i · 2^{-ℓ_i}  ≤ ε
```

（在未报告 failure 的条件下。）

## 3. 查询为何化为前缀匹配

原文：§3.1；Claim 13 末段。

- **Insert**：存 `s = h(x)[1..ℓ_i]`（若尚无更短前缀已在库中）。
- **Lookup(y)**：算 `h(y)`，问是否存在已存串 `s` 使得 `s` 是 `h(y)` 的前缀。
- **无 FN**：真成员的前缀仍在某个 `T_*` / `D_*` 中（迁移不丢数据）。
- **有界 FP**：非成员只可能“碰巧”匹配某前缀，概率由 `ℓ_i` 控制。

Claim 13 实现层查询（`i = ⌈log n⌉`）只查常数个结构：

```text
query(D_{i-1}, h(y))
query(D_i,     h(y))
query(T_{i-1}, h(y) 的前 i-1 位)
query(T_i,     h(y) 的前 i 位)
```

任一 YES → YES；全 NO → NO。

## 4. 为何按规模翻倍分阶段

原文：§1.4；§3.1（“log n stages”；第 `i` 阶段为第 `2^{i-1}+1` 次到第 `2^i` 次插入）。

| 目的 | 做法 |
|---|---|
| 空间跟当前 `n` | 容量按约 `2^i` 组织，而不是预先按未知的最终 `N` |
| 避免一次 `Θ(n)` 重建 | 每次插入只做常数步 `decrement` / `destroy` / `initialize`（文中 **10 次**） |
| 迁移中仍可查询 | 新旧 `T/D` 短暂并存；Claim 13 保证查询仍只碰常数个结构 |

**与一次性扩容的差别**：filter 已丢原键，不能按新值域重哈希全部元素；本文用“保留前缀 + 分阶段迁移”代替重建。

## 5. Day 1 说法的原文核查

| Day 1 说法 | 核查结果 | 原文位置 |
|---|---|---|
| 每次插入做 **10** 轮维护 | **属实**：`execute the following procedure for 10 times` | Claim 13 证明，插在 `insert(D_i,x)` 之后 |
| 阶段边界 / 活跃结构 | `n ∈ [2^i, 2^{i+1})` 时数据在 `D_{i-1},T_{i-1},D_i,T_i`；`n` 到达 `2^{i+1}` 时旧结构销毁、`D_{i+1},T_{i+1}` 已初始化 | Claim 13 证明段 “Clearly, for any integer n∈…” |

| 查询四个结构 | **属实** | Claim 13 末 |

## 6. 连续插入 8 个元素的示例（教学）

### 6.1 设定与证明边界

- 抽象元素：`x1 … x8`，按插入次序编号。
- **示意前缀**（教学用，非真实哈希）：假设每次写入的串长度均为对应 `ℓ_{i★}`，且两两不以对方为前缀，因而每次新插入都进入 `D_{i★}`。
- 不模拟 §5 桶内细节；只跟踪 Claim 13 层的 `T_*` / `D_*`。
- `initialize` / `destroy` 需多次调用才完成：表中写“推进中”。

**本示例可验证**：`i★` 控制流、新键进 `D_i`、查询活跃四结构、迁移去向骨架。  
**不可用于证明**：FP、failure、空间领先常数、字面常数 10、真实 `ℓ_i`、迁移位级峰值、§5 冗余。

初始（Claim 13）：已有 `T0, T1, D0=D(1,ℓ_0), D1=D(2,ℓ_1)`。  
“已初始化”的下一层可能存在，但**查询活跃**仅下表最后一列（A 审阅意见 1）。

### 6.2 逐步状态表（按 `i★`，非按半开区间归纳句）

| n | i★ | epoch | 新元素进入 | 本步维护主要推进 | 查询活跃 |
|---|---|---|---|---|---|
| 1 | 0 | 0 | `D0` ← `s1` | 无 `T_{-1}/D_{-1}`；`T1/D1` 已在初始集 | `D0,T0` |
| 2 | 1 | 1 | `D1` ← `s2` | 迁 `T0/D0`；推进 destroy / init `T2/D2` | `D0,D1,T0,T1` |
| 3 | 2 | 1 | `D2` ← `s3` | 迁 `T1/D1`；推进 destroy / init `T3/D3` | `D1,D2,T1,T2` |
| 4 | 2 | 2 | `D2` ← `s4` | 继续迁 `T1/D1` | `D1,D2,T1,T2` |
| 5 | 3 | 2 | `D3` ← `s5` | 迁 `T2/D2`；推进 destroy / init `T4/D4` | `D2,D3,T2,T3` |
| 6 | 3 | 2 | `D3` ← `s6` | 继续搬旧层 | `D2,D3,T2,T3` |
| 7 | 3 | 2 | `D3` ← `s7` | 同上 | `D2,D3,T2,T3` |
| 8 | 3 | 3 | `D3` ← `s8` | 继续；幂次边界上的存储归纳仍待统一变量后证明 | `D2,D3,T2,T3` |

### 6.3 一次插入的固定骨架（与图一致）

```text
Insert 第 n 个元素:
  i ← i★(n) = ⌈log n⌉
  s ← h(x) 的前 ℓ_i 位
  若库中尚无 s 的前缀:  insert(D_i, s)
  重复 10 次:          // 原文如此；字面配平见 issue-constant-10
    (1) 若 T_{i-1} 非空: y ← decrement(T_{i-1});
        若得到 y: insert(T_i, y∘0), insert(T_i, y∘1)
    (2) 若 D_{i-1} 非空: y ← decrement(D_{i-1});
        若得到 y:  |y|>i → insert(D_i,y) ; 否则 insert(T_i,y)
    (3) 旧结构已空未销毁 → destroy
    (4) 旧结构已销毁 → initialize(T_{i+1}), initialize(D_{i+1})
```

### 6.4 例子要说明的三点

1. **新元素**几乎总是进当前 `D_i`，不是进 `T_i`；`T` 主要由短串迁移/扩展填充。
2. **查询活跃**始终是 `D_{i-1},D_i,T_{i-1},T_i`（`i=i★`），不等于“所有已 initialize 的结构”。
3. **迁移**是后台进度，不是“第 4、8 次插入瞬间整体拷贝”。

### 6.5 迁移去向示意（短串 / 长串）

```text
从 D_{i-1} 取出 y:
  |y| > i  ──► 仍视为“长串”──► D_i
  |y| ≤ i  ──► 视为“短串”──► T_i

从 T_{i-1} 取出 y:
  ──► 写入 T_i 的两个孩子 y∘0 与 y∘1
      （真值表在更细一层上标记）
```
