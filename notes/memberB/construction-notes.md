## 2. 整体构造路线

### 2.1 元素如何被 hash

1. 取全局哈希 `h(x)`（高度独立），得到长比特串。
2. 第 `n` 次插入处于阶段 `i = ⌈log n⌉`。
3. 只把 `h(x)` 的前 `ℓ_i` 位写入结构，其中（原文式 (2)）：

```text
ℓ_i = i + log(1/ε) + log i + log log log u + 2
```

4. **直觉**：`i` 随 `n` 增大 → 前缀变长 → 每键多付约 `log log n` 量级；同时把各阶段误报预算压到可并合 ≤ `ε`。

### 2.2 为什么用 prefix 表示 / 查询为何是前缀匹配

- Filter 不存原始 `x`，只存哈希前缀。
- Lookup(`y`)：算 `h(y)`，问数据库中是否存在某个已存串 `s`，使得 `s` 是 `h(y)` 的前缀。
- 若 `y` 真的插入过：对应前缀在库中 → **无 false negative**。
- 若未插入：偶然匹配某前缀的概率由 `ℓ_i` 与并合界控制 → **FP ≤ ε**（条件：未 failure）。

### 2.3 为什么要分阶段

- 目标：空间依赖 **当前** `n`，不是事先 `N`。
- 阶段 `i`：处理第 `2^{i-1}+1` 到第 `2^i` 次插入；阶段结束时规模约翻倍。
- 每个阶段配合容量约 `m = 2^i` 的结构；同时在后台把旧容量结构迁到新容量结构。

### 2.4 扩容 / 迁移时发生什么

对第 `n` 次插入（`i = ⌈log n⌉`）：

1. 若该串尚无更短前缀已存，则 `insert(D_i, x)`（长串走 `D`；短串逻辑在迁移里进 `T`）。
2. 然后重复常数次维护（原文 10 次），大致包括：
   - 若 `T_{i-1}` 非空：`decrement` 出一个短串 `y`，写入 `T_i` 的两个扩展 `y∘0`、`y∘1`；
   - 若 `D_{i-1}` 非空：迁出一个 `y`；若 `|y| > i` 插入 `D_i`，否则插入 `T_i`；
   - 旧结构清空后 `destroy`；并 `initialize` 下一阶段的 `T_{i+1}, D_{i+1}`。
3. 当 `n` 到达 `2^{i+1}` 时：新结构已就绪，旧结构已销毁，进入下一阶段。

**关键性质（§1.4）**：迁移几乎 in-place，过程中仍可查询；总空间仍随当前 `n` succinct。

### 2.5 §5 已知容量 `D(m, ℓ)` 的一层直觉

```text
x ──► st(x) ──► main table 桶 ──► subtable
                                    ├─ adaptive prefixes（找唯一短前缀）
                                    ├─ navigators（指向 data block）
                                    └─ data blocks
                                         ├─ 已满块：静态紧凑编码
                                         └─ 最新块：动态插入 + 后台重组为静态
```

Data block 是相对 Bercea–Even 类方案的关键：把冗余从约 `O(log log u)`/键压到约 `O(log log log u)`/键，才能让 filter 的 `log log n` 项前导常数达标。

## 3. Filter 与 Dictionary 的构造关系

| | Filter (Thm 10) | Dictionary (Thm 12) |
|---|---|---|
| 存什么 | `h(x)` 的变长前缀 | 键（经随机化）+ 可选 value |
| 核心引擎 | Prefix matching (Lemma 11) | 同一套桶/块思想 + Feistel 等 |
| 空间主项 | `log(1/ε) + log log n` | `log(u/n) + v` |
| 共同点 | 分阶段、去均摊、whp 最坏 `O(1)` | 同左 |

filter 是“随机前缀上的 prefix matching”；dictionary 是“把最坏情况输入随机化后，用同类 succinct membership”。
