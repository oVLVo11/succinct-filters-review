## 1. 两个空间基准

### 1.1 已知容量

若初始化时知道容量上界 `N`，误报率为 `epsilon`，经典 filter 的空间下界是：

```text
N log(1/epsilon) bits
```

这里可以把 `log(1/epsilon)` 理解为“为了把误报压到 `epsilon`，每个 key 至少要提供多少位区分信息”。

原文位置：第 2 页；引用 [8, 23]。

### 1.2 未知最终大小

PSW 对空间随当前大小 `n` 变化的 filter 给出信息论下界：

```text
(1-o(1)) n(
    log(1/epsilon)
    + (1-O(epsilon)) log log n
)
```

它说明：unknown-size filter 除了普通误报成本外，还要额外付出接近每个 key `log log n` 位。

原文位置：摘要、第 2 页和第 4 页；引用 [28]。

“regardless of the insertion and query times” 表明这是空间层面的信息论限制，不是因为某个算法写得慢或实现得不够巧。

---

## 2. `log(1/epsilon)` 是什么成本

误报率越小，非成员越不容易与已存信息碰撞，结构就需要保存更多区分信息。

简单例子：

| `epsilon` | `log2(1/epsilon)` 约为 |
|---|---:|
| `1/2` | 1 bit |
| `1/16` | 4 bits |
| `1/1024` | 10 bits |

这张表只用于解释数量级，不代表“每个 key 独立存一段恰好这么长的比特串”。下界讨论的是整个数据结构至少需要多少总信息。

---

## 3. `log log n` 是什么成本

### 3.1 本文构造给出的直觉

不知道最终规模时，插入过程被分成约 `log n` 个翻倍阶段。非成员查询只要在任一阶段结构中误报，最终答案就会误报，所以不同阶段的误报概率必须共同受控。

在论文公式 (2) 中，第 `i` 阶段前缀长度包含 `log i`。又因为 `i≈log n`，所以每个 key 多出的长度约为：

```text
log i ≈ log log n
```

这很好地解释了本文上界中该项如何产生。

### 3.2 下界比这个直觉更强

“本文的分阶段做法要付 `log log n`”并不能推出“所有算法都必须付 `log log n`”。也许存在完全不同、不要阶段的结构。

PSW 下界排除了这种希望：只要空间必须适应当前 `n`，任何 filter 都要付近似 `log log n` 的额外信息。
本文构造中的阶段误报预算解释了上界为何出现 `log log n`；PSW 的独立信息论下界进一步证明，这一量级不是本文构造造成的浪费。

## 4. PSW 下界目前能确认到什么程度

从本论文可以确认：

1. 下界针对空间随当前 `n` 变化的 filter；
2. 下界为
   `(1-o(1))n(log(1/epsilon)+(1-O(epsilon))log log n)`；
3. 它不依赖插入和查询时间限制；
4. PSW 还给出接近下界的构造，但插入是 expected amortized constant time；
5. PSW 的 `log log n` 项只写成 `O(log log n)`，没有钉死上界领先常数；
6. 本文要解决的开放问题包括：钉死该项领先常数，并同时获得高概率最坏情况常数时间。

从本论文不能确认：

1. 下界编码论证的每一步；
2. 下界如何选择多个检查时刻或集合规模；
3. 证明如何构造误报集合；
4. 对常数 `epsilon`，`1-O(epsilon)` 中隐藏常数的精确值。

以上四点必须阅读 PSW 原论文后才能写成详细证明。

## 5. `(1-O(epsilon))` 应如何理解

PSW 下界的额外项是：

```text
(1-O(epsilon)) log log n
```

### 当 `epsilon=o(1)` 时

`epsilon` 随规模增长趋近 0，于是：

```text
1-O(epsilon) → 1
```

下界的 `log log n` 领先系数趋近 1。本文上界中对应项也是 `1·log log n`，所以摘要说本文“对所有 `epsilon=o(1)` 达到最优领先常数”。

### 当 `epsilon` 是固定常数时

例如 `epsilon=0.01`，`1-O(epsilon)` 仍然是接近 1 的常数，但仅凭大 O 记号不能断言它严格等于 1。因此本 review 不应声称本文对所有常数 `epsilon` 都证明了 `log log n` 项领先常数恰好最优。

可靠表述：

> 本文明确声称，在 `epsilon=o(1)` 时，`log log n` 项的领先常数达到最优。

原文位置：摘要；第 2 页。

## 6. 本文上界如何接近下界

正式 Theorem 10 的主体空间是：

```text
n(log(1/epsilon)+log log n+O(log log log u))
```

另有与输入无关的 `u^c` 位预计算空间。

当 `n>u^0.001` 且取 `c<0.001` 时：

```text
u^c = o(n)
```

同时：

```text
log log log u = o(log log n)
```

于是空间可以整理为非正式 Theorem 1 的形式：

```text
(1+o(1))n(log(1/epsilon)+log log n)
```

当 `epsilon=o(1)` 时，PSW 下界中 `log log n` 的系数趋近 1，因此本文上界与其领先主项吻合。

---

## 7. `n>u^0.001` 到底起什么作用

Day 1 把它猜成“可能用于桶负载或概率界”。精读正式定理后，需要修正：

- 正式 Theorem 10 的构造条件是 `n=ω(log u)` 且 `n<u`；
- `n>u^0.001` 出现在摘要和非正式 Theorem 1；
- 它最直接的作用是让低阶项相对 `n log log n` 足够小，并让 `u^c` 预计算空间在选择足够小的 `c` 后可被吸收。

所以目前更可靠的结论是：

> `n>u^0.001` 主要用于把正式空间公式简化为 `(1+o(1))` 主项，而不是正式构造唯一能运行的规模范围。

仍需注意：论文后续概率分析也以 `u` 为参数，并使用 `n<u`、有限独立哈希和桶负载界。不能进一步声称 `n>u^0.001` “完全与概率分析无关”，除非逐节检查所有参数。

---

## 8. `O(log log log u)` 会不会破坏 filter 的最优性

正式空间每个 key 还有：

```text
O(log log log u)
```

在 `n>u^0.001` 时：

```text
log n > 0.001 log u
log log n = log log u + O(1)
```

因此：

```text
log log log u / log log n → 0
```

也就是三重对数项比双重对数项增长慢得多，可以成为低阶项，不改变 `log log n` 的领先常数。

直观例子：即使 `u` 大得夸张，连续取三次对数后增长也非常慢。但这只是帮助理解，正式论证仍以上面的渐近关系为准。

---

## 9. 高概率最坏情况常数时间与空间下界并不冲突

空间下界回答：

> 至少要保存多少位信息？

时间保证回答：

> 一次操作要执行多少步？

二者是不同维度。达到最小空间不自动意味着操作快；操作快也不自动意味着空间最优。

前作存在两类取舍：

- 空间 succinct，但插入只有 expected amortized `O(1)`；
- 操作达到高概率 worst-case `O(1)`，但空间多一个常数因子。

本文的贡献是把迁移和重组拆成每次插入后的常数步，并控制迁移期间的空间峰值，从而同时逼近空间下界和获得最坏情况时间保证。

不过，“几乎原地迁移为何严格不超空间”需要结合 Claim 13 和第 5 节底层结构继续证明，不能只凭这一段直觉宣告完成。

## 10. 与正确性证明的连接

### 10.1 无 false negative

已插入 key 的哈希前缀始终保存在活跃结构中；查询检查 `D_{i-1}`、`D_i`、`T_{i-1}`、`T_i`。只要没有 failure，至少一个结构能找到该前缀。

### 10.2 false positive 不超过 `epsilon`

对非成员查询，误报来自其哈希值碰巧匹配某个已存前缀。公式 (2) 设置前缀长度，对阶段内 key 和所有阶段使用 union bound，得到总误报率小于 `epsilon`。

### 10.3 failure 不超过 `u^{-C}`

每个已知容量结构 `D_i` 的失败概率至多 `u^{-2C}`。对最多 `log u` 个阶段使用 union bound：

```text
Σ u^{-2C} ≤ u^{-C}
```

原文位置：第 12 页 Claim 13 证明。

注意：false positive 与 failure 是两种不同事件，不能把上面两种 union bound 混成一个结论。

## 14. Day 3 二轮续查：PSW 原文下界证明

新增来源：Rasmus Pagh、Gil Segev、Udi Wieder，*How to Approximate a Set without Knowing Its Size in Advance*，FOCS 2013；仓库文件 `1304.1188v2.pdf`。
精读范围：§1.3 的证明直觉、§3 The Lower Bound、Theorem 3.1、Lemma 3.2--3.4，PDF 第 3--4、7--10 页。

### 14.1 正式定理

设 unknown-size approximate membership 结构为 `D`，全集 `U` 大小为 `u`，误报率 `0<epsilon<1`。Theorem 3.1 设：

```text
n <= epsilon*u，n 充分大；
1/sqrt(n) <= alpha < 1；
gamma >= 2 为整数。
```

如果对任意长度 `m` 满足 `alpha*n < m < n` 的插入序列，结构都只使用至多 `beta*m` 位，那么：

```text
beta >= (1-1/gamma) *
        (log(1/epsilon)
         +(1-9epsilon) log log_gamma(1/alpha)
         -Theta(1)).
```

原文位置：PSW PDF 第 7 页 Theorem 3.1。

这个定理证明：如果结构要在一整个规模区间内都维持每元素至多 `beta` 位，那么 `beta` 必须包含额外双重对数项。等价地，在处理一条不太短的在线插入序列时，至少存在某个中间时刻达到该成本。

### 14.2 第一步：固定内部随机性

对固定随机串 `r` 和插入序列 `S`，令：

```text
hat(S)_r = 处理 S 后，结构回答 YES 的所有 universe 元素组成的集合；
mu(hat(S)_r) = |hat(S)_r|/u。
```

无 false negative 保证 `S` 中元素都在 `hat(S)_r` 中。误报率至多 `epsilon`，加上真实成员所占比例，得到：

```text
E_r[mu(hat(S)_r)] <= epsilon+n/u <= 2epsilon，
```

最后一步使用 `n<=epsilon*u`。由 Markov 不等式，对每个 `S`，至少一半随机串使 `mu(hat(S)_r)<4epsilon`。交换平均次序后，可固定一个随机串 `r*`，使至少一半长度 `n` 的序列满足：

```text
|S_{r*,epsilon}| >= u^n/2。
```

接下来只需对确定性结构 `B_{r*}` 做编码论证，同时仍保留指数级多的待编码序列。

原文位置：PSW PDF 第 8 页 Lemma 3.2。

### 14.3 第二步：按几何块切分序列

把序列分成连续块：

```text
S = C_1,C_2,...，其中 |C_i|=gamma^i；
S_i = 前 i 个块拼接成的前缀；
n_i = |S_i|。
```

没有漏报使正回答集合单调增长。从规模 `alpha*n` 到 `n` 之间约有 `log_gamma(1/alpha)` 个候选块，而正回答集合总测度至多 `4epsilon`。抽屉原理保证存在一个块 `C_i`，使：

```text
mu(hat(S)_i)-mu(hat(S)_{i-1})
    <= 4epsilon/(log_gamma(1/alpha)-2)。
```

Lemma 3.3 找的是“正回答集合新增测度小的块”。额外的 `log log_gamma(1/alpha)` 正是对这个小增量取对数产生的。

原文位置：PSW PDF 第 8--9 页 Lemma 3.3。

### 14.4 第三步：控制块中的旧误报

令：

```text
k = |C_i intersect hat(S)_{i-1}|，
```

即真正插入 `C_i` 前已经被回答 YES 的块内元素数。Lemma 3.4 证明，对至少 `u^n/3` 条序列有：

```text
k <= 9epsilon*|C_i|。
```

证明从整个 `U^n` 独立均匀采样以避免条件依赖，对每个足够大的块用 Chernoff 界，再对各块使用 union bound；最后与 Lemma 3.2 保留的序列集合相交。

原文位置：PSW PDF 第 9 页 Lemma 3.4、式 (3.1)。

### 14.5 第四步：用中间状态编码序列

对上述好序列，编码器写入：

1. Lemma 3.3 选出的块编号 `i`；
2. 除 `C_i` 外的全部元素，占 `(n-c_i)log u` 位，其中 `c_i=|C_i|`；
3. 一个 `c_i` 位 bitmap，标出哪 `k` 个元素在插入前已经属于正回答集合；
4. 插入 `C_i` 后的数据结构状态，共 `b_i` 位；
5. 利用块前后正回答集合，对 `C_i` 内元素做相对编码。

结构已固定为确定性结构。给出块外元素和中间状态后，可确定 `hat(S)_{i-1}` 与 `hat(S)_i`。因此：

- `c_i-k` 个新元素来自小差集 `hat(S)_i \ hat(S)_{i-1}`；
- `k` 个旧误报元素来自 `hat(S)_{i-1}`，上界时可放宽到 `hat(S)_i`。

式 (3.2) 给出的相对编码长度为：

```text
(c_i-k) log((mu(hat(S)_i)-mu(hat(S)_{i-1}))u)
+ k log(mu(hat(S)_i)u)
+ O(1)。
```

代入 Lemma 3.3、`k<=9epsilon*c_i` 和 `mu=O(epsilon)`，得到至多：

```text
c_i(log u + log epsilon
    -(1-9epsilon)log log_gamma(1/alpha)
    +O(1))。
```

原文将常数 `4` 等吸收到 `O(1)`，不能把简化后的 `log epsilon` 误读为正回答集合测度严格等于 `epsilon`。

原文位置：PSW PDF 第 9--10 页，式 (3.2) 及后续推导。

### 14.6 第五步：不可压缩性迫使状态足够大

Lemma 3.4 给出至少 `u^n/3` 条不同好序列，任何无歧义编码至少需要：

```text
log(u^n/3)=n log u-O(1)
```

位。将编码各部分求和并与计数下界比较，得到：

```text
b_i >= c_i(
          log(1/epsilon)
          +(1-9epsilon)log log_gamma(1/alpha)
          -O(1))。
```

几何块还满足 `c_i/n_i >= 1-1/gamma`。结合假设 `b_i<=beta*n_i`，即得 Theorem 3.1。证明本质是：若每个中间状态都太小，就能把至少 `u^n/3` 条序列编码得短于计数下界，产生矛盾。

### 14.7 如何推出 `n log log n`

原文取：

```text
alpha=1/sqrt(n)，
gamma=2^{(log n)^eta}，其中 0<eta<1。
```

于是：

```text
1-1/gamma = 1-o(1)，
log_gamma(1/alpha)=Theta((log n)^{1-eta})，
log log_gamma(1/alpha)=(1-eta)log log n-O(1)。
```

从而推出 Theorem 1.1：

```text
(1-o(1))n log(1/epsilon)+Omega(n log log n)。
```

若让 `gamma` 更缓慢地增长，可同时保持 `1/gamma=o(1)` 和：

```text
log log_gamma(1/alpha)=log log n-o(log log n)。
```

结合 `(1-9epsilon)`，额外项系数为 `1-O(epsilon)-o(1)`。因此 Liu--Yin--Yu 将 PSW 参数化结果概括为：

```text
(1-o(1))n(log(1/epsilon)+(1-O(epsilon))log log n)。
```

严谨表述应注明：PSW Theorem 1.1 字面以 `Omega(n log log n)` 陈述；更细领先系数来自 Theorem 3.1 的参数化形式。

### 14.8 为什么该下界与操作时间无关

PSW 允许结构免费访问只读随机比特数组，也不限制 membership 算法时间；证明只读取某时刻的结构状态并进行信息论编码。因此即使时间无限宽松，该空间下界仍成立。
